## Purpose

This runbook provides the procedures required to investigate and respond to security alerts generated within the Northwind Technologies environment.

This runbook applies to alerts generated by Wazuh, Sysmon, Windows Event Logs, Active Directory, and future security monitoring systems.

---

## Symptoms

- Wazuh generates a high- or medium-severity alert.
- Unusual login activity is detected.
- Multiple failed authentication attempts occur.
- Unexpected process execution is observed.
- Network connections appear suspicious.
- Malware indicators are identified.
- Security dashboards display critical events.
- Analysts receive:
    - "Authentication failure detected."
    - "Suspicious process execution."
    - "Multiple failed logins."
    - "Unauthorized access attempt."
    - "Potential malware activity."

## Investigation

1. Verify analyst authorization.
2. Open the Wazuh dashboard.
3. Identify:
    - Alert severity
    - Alert rule ID
    - Source endpoint
    - Username
    - Timestamp
4. Review endpoint status: ```powershell Get-Service WazuhSvc```
5. Review Windows Security logs: ```powershell eventvwr.msc```
    Navigate:
        Applications and Services Logs
            └── Microsoft
                └── Windows
                    └── Sysmon
                        └── Operational
6. Review Sysmon events.
7. Review recent authentication activity: ```powershell Get-WinEvent -LogName Security -MaxEvents 50```
8. Review network connections: ```powershell netstat -ano```
9. Review running processes: ```powershell tasklist```
10. Correlate findings in Wazuh.

## Common Alert Types

| Alert                 | Investigation Focus     |
| --------------------- | ----------------------- |
| Failed Logins         | Brute-force attempts    |
| New Process Execution | Suspicious applications |
| PowerShell Activity   | Script abuse            |
| DNS Queries           | Command-and-control     |
| File Creation         | Malware activity        |
| Service Changes       | Persistence             |

---

## Resolution

1. Determine whether the alert is:
    - True Positive
    - False Positive
    - Benign Activity
2. Document findings.
3. Collect:
    - Username
    - Hostname
    - IP address
    - Event ID
    - Rule ID
4. Isolate affected endpoints if required.
5. Disable compromised accounts if necessary.
6. Escalate critical incidents.
7. Create or update an osTicket incident.
8. Assign incident severity.
    - Low
    - Medium
    - High
    - Critical
9. Continue monitoring.
10. Record remediation actions.

---

## Validation

- Alert source is identified.
- Scope of impact is understood.
- Malicious activity is confirmed or ruled out.
- Incident documentation is complete.
- Wazuh continues monitoring successfully.
- Ticket has been updated.

---

## Escalation Criteria

Escalate if:
- Privileged accounts are involved.
- Multiple endpoints are affected.
- Malware is suspected.
- Data exfiltration indicators exist.
- Domain controllers are impacted.
- Security monitoring fails.
- Business operations are disrupted.
