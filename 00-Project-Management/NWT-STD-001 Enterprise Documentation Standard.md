**Version:** 1.0

**Status:** Approved

**Owner:** Information Technology Department

---

# Purpose

The purpose of this standard is to establish a consistent format for all technical documentation produced within the Northwind Technologies Enterprise IT Support Lab.

Consistent documentation improves readability, maintainability, collaboration, and professionalism while accurately reflecting enterprise documentation practices.

This standard applies to all documentation contained within the Enterprise IT Support Lab repository.

---

# Document Header Standard

Every major document begins with the following information:

* Northwind Technologies
* Enterprise IT Support Lab
* Document Title
* Document ID
* Version
* Status
* Author
* Last Updated

Example:

```text
Northwind Technologies

Enterprise IT Support Lab

Document:
Project Charter

Document ID:
NWT-PM-001

Version:
1.0

Status:
Approved

Author:
Brandon J. Cooper

Last Updated:
June 2026
```

---

# Required Sections

Every major document should include the following sections when applicable:

* Purpose
* Scope
* Background
* Business Justification
* Implementation
* Validation
* Lessons Learned
* Related Documentation
* Revision History

---

# Naming Standards

Documentation shall use Title Case.

Examples:

## Project Management
* NWT-PM-001  Project Charter
* NWT-PM-002  Roadmap
* NWT-PM-003  Milestones
* NWT-PM-004  Risk Register
* NWT-PM-005  Change Management

## Enterprise Documentation
* NWT-ENT-001 Company Profile
* NWT-ENT-001 Company Profile
* NWT-ENT-002 Enterprise Architecture
* NWT-ENT-003 Network Diagram
* NWT-ENT-004 Server Inventory
* NWT-ENT-005 IP Address Plan
* NWT-ENT-006 Naming Standards

## Standard Operating Procedures

* NWT-SOP-001 Password Reset
* NWT-SOP-002 Unlock User
* NWT-SOP-003 Create New User
* NWT-SOP-004 New Hire
* NWT-SOP-005 Termination

Avoid abbreviations unless they are industry standard.

---

# Markdown Standards

Use:

* Level 1 heading for the document title.
* Level 2 headings for primary sections.
* Level 3 headings for subsections.
* Tables for structured information.
* Bullet lists for procedures.
* Numbered lists for sequential tasks.
